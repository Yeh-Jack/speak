D. SSE (Server-Sent Events) - A server-push technology where the server sends data to the client over HTTP, one-way communication.
   Use case for this project:
    - The single HTTP request stays open and the server pushes data chunks through it. No new connection needed.
    - Backend sends tokens one-by-one as LLM generates them
    - Frontend displays text in real-time (like ChatGPT streaming)
    - Simple to implement with FastAPI's StreamingResponse
   How it works:
   Client (Vue) ───HTTP request───▶ Backend
        ▲                        │
        │                        ▼
        └───── SSE stream ◀───── LLM generates tokens
   Alternative: WebSockets (bidirectional, more complex) - not needed since only server→client streaming is required.
   Key Design Pattern: Running a Blocking Generator in Async Context
   The challenge: llama-cpp-python's create_chat_completion with stream=True is a blocking iterator
   that must run in a thread to avoid blocking the event loop.

  1. Main async code calls loop.run_in_executor to run _stream_tokens in thread pool
  2. _stream_tokens is a generator that yields tokens from the LLM
  3. The while loop continuously pulls tokens using _get_next (also run in executor)
  4. Tokens are yielded back to the async caller
  I'll create a simple ASCII/text diagram.
  ┌─────────────────────────────────────────────────────────────┐
  │                      Async Caller                           │
  └─────────────────────┬───────────────────────────────────────┘
                        │ yields token
                        ▼
  ┌─────────────────────────────────────────────────────────────┐
  │  while True:                                                │
  │    token = await loop.run_in_executor(None, _get_next, gen) │
  │    if token is None: break                                  │
  │    yield token                                              │
  └─────────────────────┬───────────────────────────────────────┘
                        │ run in thread pool
                        ▼
  ┌─────────────────────────────────────────────────────────────┐
  │  Thread Pool Executor (None = default)                      │
  │  ┌────────────────────────────────────────────────────────┐ │
  │  │  _get_next(gen)         _stream_tokens()               │ │
  │  │       │                        │                       │ │
  │  │       │                        ▼                       │ │
  │  │       │                 ┌──────────────┐               │ │
  │  │       │                 │  LLM Model   │               │ │
  │  │       │                 │  (blocking)  │               │ │
  │  │       │                 └──────┬───────┘               │ │
  │  │       │                        │                       │ │
  │  │       │◄───────── yields ──────┘                       │ │
  │  └───────┴────────────────────────────────────────────────┘ │
  └─────────────────────────────────────────────────────────────┘
  Flow:
  1. _stream_tokens runs in thread pool, blocking on LLM
  2. _get_next pulls next token from generator (also in thread pool)
  3. Main loop runs in async context, yielding tokens to caller
  4. When StopIteration occurs, _get_next returns None → break

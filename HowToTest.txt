B. SSE (Server-Sent Events) - A server-push technology where the server sends data to the client over HTTP, one-way communication.
   Use case for this project:
    - The single HTTP request stays open and the server pushes data chunks through it. No new connection needed.
    - Backend sends tokens one-by-one as LLM generates them
    - Frontend displays text in real-time (like ChatGPT streaming)
    - Simple to implement with FastAPI's StreamingResponse
   How it works:
   Client (Vue) ───HTTP request───▶ Backend
        ▲                        │
        │                        ▼
        └───── SSE stream ◀───── LLM generates tokens
   Alternative: WebSockets (bidirectional, more complex) - not needed since only server→client streaming is required.
